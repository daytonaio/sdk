Core workspace functionality for Daytona.

Provides the main Workspace class that coordinates file system,
Git, process execution, and LSP functionality. It serves as the central point
for managing and interacting with Daytona workspaces.

**Example**:

  Basic workspace operations:
```python
from daytona_sdk import Daytona
daytona = Daytona()
workspace = daytona.create()

# File operations
workspace.fs.upload_file("/workspace/test.txt", b"Hello, World!")
content = workspace.fs.download_file("/workspace/test.txt")

# Git operations
workspace.git.clone("https://github.com/user/repo.git")

# Process execution
response = workspace.process.exec("ls -la")
print(response.result)

# LSP functionality
lsp = workspace.create_lsp_server("python", "/workspace/project")
lsp.did_open("/workspace/project/src/index.ts")
completions = lsp.completions("/workspace/project/src/index.ts", Position(line=10, character=15))
print(completions)
```
  

**Notes**:

  The workspace must be in a 'started' state before performing operations.

Represents a Daytona workspace.

A workspace provides file system operations, Git operations, process execution,
and LSP functionality. It serves as the main interface for interacting with
a Daytona workspace.

**Attributes**:

- `id` _str_ - Unique identifier for the workspace.
- `instance` _WorkspaceInstance_ - The underlying workspace instance.
- `workspace_api` _WorkspaceApi_ - API client for workspace operations.
- `toolbox_api` _ToolboxApi_ - API client for toolbox operations.
- `code_toolbox` _WorkspaceCodeToolbox_ - Language-specific toolbox implementation.
- `fs` _FileSystem_ - File system operations interface.
- `git` _Git_ - Git operations interface.
- `process` _Process_ - Process execution interface.

[[view_source]](https://github.com/daytonaio/sdk/blob/be977152ed0ca945a93b55bd011bfbb8e78dad53/packages/python/src/daytona_sdk/workspace.py#L293)

Start the workspace.

This method starts the workspace and waits for it to be ready.

**Arguments**:

- `timeout` _Optional[float]_ - Maximum time to wait in seconds. 0 means no timeout.
  Defaults to 60-second timeout.
  

**Raises**:

- `ValueError` - If timeout is negative.
- `Exception` - If workspace fails to start or times out.
  

**Example**:

```python
workspace = daytona.get_current_workspace("my-workspace")
workspace.start(timeout=40)  # Wait up to 40 seconds
print("Workspace started successfully")
```

#### Workspace.set\_autostop\_interval

[[view_source]](https://github.com/daytonaio/sdk/blob/be977152ed0ca945a93b55bd011bfbb8e78dad53/packages/python/src/daytona_sdk/workspace.py#L406)

Set the auto-stop interval for the workspace.

The workspace will automatically stop after being idle (no new events) for the specified interval. Events include any state changes or interactions with the workspace through the sdk.
Interactions using Workspace Previews are not included.

**Arguments**:

- `interval` _int_ - Number of minutes of inactivity before auto-stopping.
  Set to 0 to disable auto-stop. Defaults to 15.
  

**Example**:

```python
# Auto-stop after 1 hour
workspace.set_autostop_interval(60)
# Or disable auto-stop
workspace.set_autostop_interval(0)
```

[[view_source]](https://github.com/daytonaio/sdk/blob/be977152ed0ca945a93b55bd011bfbb8e78dad53/packages/python/src/daytona_sdk/workspace.py#L77)

Structured information about a workspace.

This class provides detailed information about a workspace's configuration,
resources, and current state.

**Attributes**:

- `id` _str_ - Unique identifier for the workspace.
- `name` _str_ - Display name of the workspace.
- `image` _str_ - Docker image used for the workspace.
- `user` _str_ - OS user running in the workspace.
- `env` _Dict[str, str]_ - Environment variables set in the workspace.
- `labels` _Dict[str, str]_ - Custom labels attached to the workspace.
- `public` _bool_ - Whether the workspace is publicly accessible.
- `target` _str_ - Target environment where the workspace runs.
- `resources` _WorkspaceResources_ - Resource allocations for the workspace.
- `state` _str_ - Current state of the workspace (e.g., "started", "stopped").
- `error_reason` _Optional[str]_ - Error message if workspace is in error state.
- `snapshot_state` _Optional[str]_ - Current state of workspace snapshot.
- `snapshot_state_created_at` _Optional[datetime]_ - When the snapshot state was created.
  

**Example**:

```python
workspace = daytona.create()
info = workspace.info()
print(f"Workspace {info.name} is {info.state}")
print(f"Resources: {info.resources.cpu} CPU, {info.resources.memory} RAM")
```
